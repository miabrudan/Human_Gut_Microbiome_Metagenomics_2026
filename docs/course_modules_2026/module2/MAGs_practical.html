1 Module 2 – Practical Exercise

1.1 Exploring Metagenome-Assembled Genomes (MAGs)

1.2 Exercise overview

In this exercise we will explore the outputs of a genome-resolved metagenomics workflow.

Running a complete metagenomic assembly and binning pipeline can take several hours on high-performance computing systems. For training purposes, the heavy computational steps have already been completed.

Participants will therefore focus on examining the outputs of the MAG reconstruction workflow, including:

  • assembled contigs

  • genome bins

  • refined MAGs

  • MAG quality metrics

This practical session will help participants understand how MAGs are generated and how their quality is evaluated.

1.3 Dataset Structure

The dataset provided for this exercise contains the results of a completed metaWRAP workflow.

1.4 Input data

For this exercise we have used pregenerated quality-controlled paired-end reads in Module 1 to assemble raw metagenome assembled genomes (MAGs).

1.5 Exercise 1 – Inspect the Assembly

First, we examine the assembly output.

Navigate to the assembly directory:

cd assembly

Check the number of contigs (FASTA sequences):

grep -c ">" SRR30598619_final_assembly.fasta
or
grep ">" SRR30598619_final_assembly.fasta | wc -l

This command counts the number of contigs produced during assembly.

1.5.1 Inspect contig lengths

To view the first few contigs:

head SRR30598619_final_assembly.fasta

1.5.1.1 Questions

  1. How many contigs were produced?

  2. Are all contigs the same length?

In real datasets, contig sizes can vary from a few hundred base pairs to hundreds of kilobases.

1.6 Exercise 2 – Explore Genome Bins

Next, examine the bins produced by the binning algorithms.

Navigate to the binning directory:

cd ../binning

List the bins produced by MetaBAT2:

ls metabat2_bins

Example output:

bin.1.fa
bin.2.fa
bin.3.fa

Each file represents a candidate genome bin.

1.6.0.1 Inspect one bin

Open a bin file:

head metabat2_bins/bin.1.fa

Each bin contains multiple contigs predicted to originate from the same genome.

1.7 Exercise 3 – Examine Refined MAGs

Now examine the refined bins produced by metaWRAP.

Navigate to the refinement directory:

cd ../refinement

List the final MAGs:

ls metawrap_bins

Each file corresponds to a refined MAG.

These genomes represent the best-quality bins after combining results from multiple binning algorithms.

1.8 Exercise 4 – Evaluate MAG Quality

Next, inspect the MAG quality statistics (estimated with CheckM).

cat metawrap_bins.stats

Example output:

bin      completeness    contamination
bin.10.orig       61.76         0.892
bin.11.orig       92.00         1.582
bin.12.orig       95.30         2.873

1.9 Interpreting the Results

Use the following guidelines:

MAG Quality Completeness Contamination
High-quality >90% <5%
Medium-quality >70% <10%
Low-quality <50% variable

1.9.0.1 Questions:

  1. How many MAGs meet high-quality criteria?

  2. Which MAG has the highest completeness?

  3. Which MAG shows the highest contamination?

1.10 Exercise 5 – Count Contigs per MAG

To see how many contigs make up each MAG:

grep -c ">" metawrap_bins/SRR30598619_bin.10.orig.fa

MAGs usually consist of multiple contigs rather than a single continuous sequence.

2 Discussion

  1. Why do MAGs typically consist of multiple contigs rather than a single chromosome?

  2. Why might some bins show contamination?

  3. How might sequencing depth influence MAG completeness?

  4. What additional steps could improve genome reconstruction?

2.0.1 Key Takeaways

In this exercise you:

  • inspected metagenomic assembly outputs

  • explored genome bins generated by binning algorithms

  • examined refined metagenome-assembled genomes

  • interpreted MAG completeness and contamination metrics

These steps illustrate how genome-resolved metagenomics reconstructs microbial genomes directly from shotgun sequencing data.
