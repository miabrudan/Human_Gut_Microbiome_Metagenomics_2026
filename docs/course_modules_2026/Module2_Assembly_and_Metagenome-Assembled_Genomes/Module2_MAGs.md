# Introduction

After quality control and host decontamination (covered in **Module 1**), sequencing reads are ready for downstream analyses. At this stage, an important question arises:

**Can we move from millions of short DNA fragments to reconstructed microbial genomes?**

Genome-resolved metagenomics aims to answer this question. Instead of only identifying *which taxa are present* in a microbial community, this approach attempts to reconstruct fragments of the actual genomes of the organisms in the sample. These reconstructed draft genomes are known as **Metagenome-Assembled Genomes (MAGs)**.

The MAG reconstruction workflow can be summarized as:

1. Short sequencing **reads** are generated through shotgun sequencing.
2. Reads are **assembled** into longer sequences called **contigs**.
3. Contigs likely originating from the same organism are grouped into **bins**.
4. Bins are refined to produce **metagenome-assembled genomes (MAGs)**.
5. MAG quality is evaluated using metrics such as **completeness** and **contamination**.

Genome-resolved metagenomics therefore enables researchers to:

- recover genomes of uncultured microorganisms
- identify previously unknown microbial species
- link metabolic pathways to specific organisms
- study microbial population structure and evolution
- expand microbial reference genome catalogues

In this module, participants will learn the conceptual principles and computational workflow used to reconstruct MAGs using the **metaWRAP pipeline**.

# Key concepts

## Reads
Reads are short DNA sequences generated by sequencing platforms. In shotgun metagenomic sequencing, DNA from all organisms present in a sample is fragmented and sequenced simultaneously.

Reads typically range from **100–300 base pairs** in length.

Example reads:

```text
ATCGTACGTA
CGTACGTAGC
TACGTAGCTA
```
Because microbial genomes are much larger than individual reads, computational assembly is required to reconstruct longer genomic sequences.

## What is assembly?
**Assembly** is the process of reconstructing longer DNA sequences by identifying overlaps between short sequencing reads.

Assembly algorithms compare reads, detect overlapping regions, and merge reads that likely originate from the same genomic region.

For example:
```text
Read 1: ATCGTACGTA
Read 2:     TACGTAGC
Read 3:         GTAGCTA
```

In metagenomic datasets this process is challenging because reads originate from **many different genomes simultaneously**.


## Contigs
A **contig** (contiguous sequence) is a continuous DNA sequence reconstructed from overlapping reads during assembly.

Example: 
```text
ATCGTACGTAGCTA
```
Contigs represent reconstructed fragments of microbial genomes. Because metagenomic assembly is complex, most genomes are represented by **multiple contigs rather than a single complete sequence**.

---

## Scaffolds
A **scaffold** is a structure formed by linking multiple contigs together using paired-end read information.

Example:
```text
Contig1 — gap — Contig2 — gap — Contig3
```
Scaffolds represent larger genomic regions but may still contain unresolved gaps.

---

## Bins

After assembly, the dataset typically contains thousands of contigs originating from many organisms.

A **bin** is a group of contigs predicted to originate from the same organism. Each bin therefore represents a **candidate draft genome**.

---

## Genome binning

**Genome binning** is the computational process of grouping contigs into bins that correspond to individual microbial genomes.

Binning algorithms rely on several features:

- **GC content**
- **k-mer or tetranucleotide frequency patterns**
- **sequencing coverage**
- **paired-read linkage**

By combining these signals, binning algorithms cluster contigs with similar sequence composition and abundance patterns.

---

## Metagenome-Assembled Genomes (MAGs)

A **Metagenome-Assembled Genome (MAG)** is a genome reconstructed from metagenomic sequencing data through assembly, binning, and refinement.

MAGs are usually incomplete and consist of multiple contigs, but they provide valuable genomic information for organisms that may not be cultivable.

---

# Why assembly matters in metagenomics

Read-based taxonomic profiling tools (which will be discussed in **Module 4**) such as:

- MetaPhlAn
- Kraken2

can identify which taxa are present in a sample but do not reconstruct genomes.

Assembly-based approaches enable researchers to:

1. reconstruct draft microbial genomes directly from environmental samples
2. discover novel microbial lineages because many microbial species cannot be cultured in laboratory conditions
3. associate genes and pathways with specific organisms
4. perform comparative genomics
5. expand microbial reference genome databases used for taxonomic and functional profiling

# The metaWRAP pipeline
metaWRAP is a modular pipeline designed for **genome-resolved metagenomic analysis**.

> GitHub repository: https://github.com/bxlab/metaWRAP

metaWRAP integrates several commonly used tools including **MEGAHIT, MetaBAT2, MaxBin2, CONCOCT, and CheckM** and allows each step to be run independently. See the manual for this step here [Module2_MAGs_metaWRAP.md](Module2_MAGs_metaWRAP.md).

In this module we demonstrate three core modules:

```text
metawrap assembly
metawrap binning
metawrap bin_refinement
```

These modules perform:
- assembly of sequencing reads
- binning of assembled contigs
- refinement and quality evaluation of genome bins

## Software requirements
The following software must be installed:
- **metaWRAP** – modular pipeline for genome-resolved metagenomics workflows  
- **MEGAHIT** – fast de Bruijn graph assembler optimized for metagenomic datasets  
- **MetaBAT2** – binning algorithm that clusters contigs based on sequence composition and coverage  
- **MaxBin2** – probabilistic binning tool that assigns contigs to genomes using marker genes and abundance patterns  
- **CONCOCT** – binning algorithm that clusters contigs based on coverage and nucleotide composition  
- **CheckM** – tool for assessing genome completeness and contamination using lineage-specific marker genes  
- **samtools** – toolkit for manipulating sequencing alignment files  
- **bowtie2** – read alignment tool used to map sequencing reads to assembled contigs

## Step 1: Metagenomic assembly
The first step in genome-resolved metagenomics is assembling sequencing reads into longer sequences.

metaWRAP performs this step using the **MEGAHIT assembler**.

### Assembly command
```bash
metawrap assembly \
    -1 DEHOSTED_READS/SRR30598619_dehosted.1.fastq.gz \
    -2 DEHOSTED_READS/SRR30598619_dehosted.2.fastq.gz \
    -o assembly_output \
    -t 16 \
    -m 64
```

### Parameter explanation
- `-1` forward reads  
- `-2` reverse reads  
- `-o` output directory  
- `-t` number of CPU threads  
- `-m` memory in GB  

### Output
```text
assembly_output/final_assembly.fasta
```

This file contains the assembled contigs.

## Step 2: Genome binning
Following assembly, contigs must be partitioned into bins representing candidate genomes.

metaWRAP can run multiple binning algorithms:

- MetaBAT2
- MaxBin2
- CONCOCT

Using multiple binning tools improves genome recovery.

### Binning command

```bash
metawrap binning \
    -o binning_output \
    -t 16 \
    -a assembly_output/final_assembly.fasta \
    --metabat2 \
    --maxbin2 \
    --concoct \
    DEHOSTED_READS/SRR30598619_dehosted.1.fastq.gz \
    DEHOSTED_READS/SRR30598619_dehosted.2.fastq.gz
```

### Output

```text
binning_output/
    metabat2_bins/
    maxbin2_bins/
    concoct_bins/
```

Each directory contains bins produced by the corresponding algorithm.

# Step 3: Bin refinement
Different binning algorithms often produce overlapping or partially inconsistent bins. The **bin_refinement** module compares these bins and selects the best versions based on genome quality metrics.

During this process, **CheckM is automatically used to evaluate completeness and contamination**.

### Refinement command

```bash
metawrap bin_refinement \
    -o refined_bins \
    -t 16 \
    -A binning_output/metabat2_bins \
    -B binning_output/maxbin2_bins \
    -C binning_output/concoct_bins \
    -c 70 \
    -x 10
```
### Parameter explanation
- `-A` is MetaBAT2 bins
- `-B` is MaxBin2 bins
- `-C` is CONCOCT bins
- `-c` is Minimum completeness threshold
- `-x` is Maximum contamination threshold 

### Output

```text
refined_bins/metawrap_bins/
```

These refined bins represent the candidate **MAGs**.


## Step 4: Quality assessment with CheckM
The quality of MAGs must be evaluated before further analysis. CheckM estimates genome quality based on lineage-specific marker genes.

Two key metrics are reported:
1. **Genome completeness** -the proportion of expected marker genes detected in the genome.
2. **Genome contamination** - evidence that sequences from multiple organisms are present in the same bin.

Example output:
```code
Bin_ID      Completeness    Contamination
bin.1       95.3%           3.2%
bin.2       87.4%           4.5%
bin.3       62.0%           7.8%
```

MAG quality is typically interpreted using MIMAG-style thresholds as shown below:

| MAG Quality    | Completeness | Contamination |
|----------------|--------------|---------------|
| High-quality   | >90%         | <5%           |
| Medium-quality | >70%         | <10%          |
| Low-quality    | <50%         | variable      |

High-quality MAGs are suitable for:
- phylogenetic analysis
- metabolic reconstruction
- comparative genomics
